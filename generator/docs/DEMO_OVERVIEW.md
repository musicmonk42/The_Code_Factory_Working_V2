# Demo Overview

This handbook introduces the **investor demo** for the AI README-to-App Code Generator, a platform that automates the generation of deployment configurations (e.g., Dockerfiles) from natural language inputs. The demo highlights the platform’s core capability—taking a simple app description, generating a production-ready Dockerfile, validating it, and saving it to disk with full observability—for investors to prove the platform works.

---

## 🎯 Purpose

The demo showcases how the platform:

1. **Ingests a simple app description** (e.g., “Create a Python Flask app, expose port 8080”)
2. **Generates a context-rich prompt** using `deploy_prompt.py`
3. **Calls an LLM provider** (e.g., xAI Grok or local Ollama) via `deploy_llm_call.py`
4. **Normalizes and enriches the output** with `deploy_response_handler.py`
5. **Validates the Dockerfile** with `deploy_validator.py`
6. **Saves the result** to `demo_output/Dockerfile` using `file_utils.py`
7. **Displays investor-friendly output** using [rich](https://github.com/Textualize/rich) and logs metrics for observability

*The demo avoids complex features (like ensemble mode or streaming) to ensure reliability and clarity.*

---

## 🗂️ Scope

- **Input:** A text description (e.g., “Create a Python Flask app, expose port 8080”)
- **Output:** A validated Dockerfile saved to `demo_output/Dockerfile`
- **Provider:** Uses `GrokProvider` (requires `GROK_API_KEY`) or `LocalProvider` (Ollama, no API key)
- **Core Components:**
  - `deploy_prompt.py`: Generates prompts with Jinja2 templates
  - `deploy_llm_call.py`: Orchestrates LLM calls and provider selection
  - `deploy_response_handler.py`: Normalizes and enriches outputs
  - `deploy_validator.py`: Validates configs with hadolint and trivy
  - `file_utils.py`: Saves outputs with compliance metadata
  - `observability_utils.py`, `runner_logging.py`: Logging and metrics

- **Observability:** Console logs, Prometheus metrics, and OpenTelemetry traces
