AI README-to-App Code Generator

The AI README-to-App Code Generator is a cutting-edge platform that automates the generation of production-ready deployment configurations (e.g., Dockerfiles, Helm charts, Terraform scripts) from natural language inputs like READMEs or app descriptions. Powered by advanced LLMs (OpenAI, xAI Grok, Google Gemini, Anthropic Claude, and local models like Ollama), it offers unparalleled reliability, security, and observability, making it the greatest orchestration system for DevOps automation.
Designed for developers, DevOps engineers, and enterprises, this platform streamlines CI/CD pipeline creation with features like self-healing, ensemble voting, and AI-driven prompt optimization. It integrates seamlessly with modern observability tools (Prometheus, OpenTelemetry) and supports extensible plugins for new providers and formats.
🌟 Features

Multi-Provider LLM Orchestration: Supports OpenAI, Grok, Gemini, Claude, and local LLMs with dynamic provider selection based on latency, cost, and quality.
Self-Improving Prompts: Uses a meta-LLM feedback loop to optimize prompts based on previous runs, ensuring high-quality outputs.
Ensemble Mode: Combines responses from multiple providers with consensus voting and diff reporting for maximum accuracy.
Advanced Observability: Prometheus metrics, OpenTelemetry tracing, and structured logging for real-time monitoring.
Security & Compliance: Prompt scrubbing, security scanning (Trivy), and compliance tagging (GDPR, CCPA, HIPAA).
Extensibility: Hot-reloadable plugins for providers, validators, and format handlers, with custom hooks for flexibility.
Streaming Support: Real-time chunked output with metrics and cancellation support.
Validation & Auto-Fix: Validates configs (Docker, Helm) with hadolint and trivy, offering LLM-driven auto-correction.
CLI & API: FastAPI-based API and CLI for easy integration and debugging.

🚀 Quick Start
Get started with a minimal setup to generate a Dockerfile from a simple app description.
Prerequisites

Python: 3.9 or higher
Dependencies: Install via requirements.txt
Optional Tools: docker, hadolint, trivy for validation
LLM Access: At least one API key (e.g., GROK_API_KEY) or a local LLM server (e.g., Ollama)
Template Directory: Create deploy_templates/docker_default.jinja (see Setup)

Installation

Clone the Repository:
git clone https://github.com/xai-org/readme-to-app.git
cd readme-to-app


Install Dependencies:
pip install -r requirements.txt

Requirements include: aiohttp, prometheus_client, opentelemetry-api, opentelemetry-sdk, opentelemetry-exporter-otlp-proto-grpc, aiofiles, rich, pyyaml, ruamel.yaml, hcl2, sentence-transformers, tiktoken, fastapi, uvicorn.

Set Up Environment:
export GROK_API_KEY="your-grok-api-key"
# OR run Ollama locally:
docker run -d -p 11434:11434 ollama/ollama

Create Template Directory:
mkdir deploy_templates
echo -e "Generate a production-grade Dockerfile for these files: {{ files | join(', ') }}.\nInclude all required build, environment, network, and security settings.\nConfiguration must be safe, efficient, and ready for CI/CD.\nAdditional instructions: {{ instructions | default('') }}" > deploy_templates/docker_default.jinja

Demo Script
The demo script (demo_investor.py) is included in the repository and demonstrates the platform’s core functionality:
import asyncio
from pathlib import Path
from utils.file_utils import save_files_to_output
from providers.grok_provider import GrokProvider
from deploy_llm_call import DeployLLMOrchestrator
from deploy_prompt import DeployPromptAgent
from deploy_response_handler import handle_deploy_response
from deploy_validator import validate_deploy_configs
from rich.console import Console
from rich.panel import Panel

console = Console()

async def run_investor_demo():
    app_description = "Create a Python Flask web app, expose port 8080, and include a basic health check endpoint."
    console.print(Panel(f"Input Description: {app_description}", title="Step 1: Input", border_style="green"))

    prompt_agent = DeployPromptAgent(repo_path=".")
    prompt_data = await prompt_agent.build_deploy_prompt(target="docker", files=["app.py"], instructions=app_description)
    console.print(Panel(f"Generated Prompt:\n{prompt_data['prompt'][:200]}...", title="Step 2: Prompt", border_style="blue"))

    orchestrator = DeployLLMOrchestrator()
    response = await orchestrator.generate_config(prompt_data["prompt"], model="grok-4")
    console.print(Panel(f"Generated Dockerfile:\n{response['config']}", title="Step 3: Generated Output", border_style="cyan"))

    handled_response = await handle_deploy_response(response["config"], output_format="dockerfile")
    console.print(Panel(f"Enriched Dockerfile:\n{handled_response['config']}", title="Step 4: Enriched Output", border_style="magenta"))

    validation_result = await validate_deploy_configs(handled_response["config"], target="docker")
    console.print(Panel(f"Validation Result: {validation_result['report']}", title="Step 5: Validation", border_style="magenta"))

    output_dir = Path("demo_output")
    await save_files_to_output({"Dockerfile": handled_response["config"]}, output_dir)
    console.print(Panel(f"Dockerfile saved to {output_dir}/Dockerfile", title="Step 6: Output", border_style="green"))

if __name__ == "__main__":
    asyncio.run(run_investor_demo())

🔒 Security & Compliance

Prompt Scrubbing: Removes sensitive data (API keys, PII) using regex patterns.
Security Scanning: Uses trivy to scan configs for vulnerabilities.
Compliance Tagging: Adds GDPR, CCPA, and HIPAA metadata via file_utils.py.
Audit Logging: Tamper-evident logs with cryptographic signatures (audit_crypto.py).

📊 Observability

Metrics: Prometheus metrics (deploy_calls_total, deploy_latency_seconds, etc.) exposed at /metrics.
Tracing: OpenTelemetry traces for all LLM calls and validations.
Logging: Structured logs with run IDs, scrubbed for sensitive data.
Notifications: Slack alerts for circuit breaker events (configure SLACK_WEBHOOK).

🧪 Testing
Run the full test suite:
python -m unittest discover

The platform includes unit, integration, and property-based tests (via hypothesis) for providers, handlers, validators, and orchestration.
🛠 Troubleshooting

Logs: Check logs for run_id to trace issues.
Metrics: Monitor /metrics for latency, errors, and provider health.
Health Checks: Use GET /health to verify provider status.
Circuit Breakers: Reset via provider’s reset_circuit() if disabled.
Debugging: Use --test flag for unit tests or --server for API debugging.

Common Issues

Missing API Key: Ensure GROK_API_KEY, OPENAI_API_KEY, etc., are set.
Local LLM Failure: Verify Ollama is running (curl http://localhost:11434).
Validation Errors: Install hadolint and trivy for full validation.
Template Not Found: Create deploy_templates/docker_default.jinja.
