# OmniCore Omega Pro Engine

> **Quick Start**: See [QUICKSTART.md](../QUICKSTART.md) for platform-wide setup, or use `make run-omnicore` from the root directory.

## Quick Commands

```bash
# From root directory
make run-omnicore            # Start OmniCore Engine API
make test-omnicore           # Run OmniCore tests
docker-compose up omnicore   # Run with Docker

# From omnicore_engine directory
python -m uvicorn fastapi_app:app --host 0.0.0.0 --port 8000 --reload  # Start API
python -m omnicore_engine.cli --help                                    # CLI help
pytest tests/ -v                                                        # Run tests
```

## Overview

The OmniCore Omega Pro Engine is a modular, extensible orchestration and simulation framework for AI-driven applications, built for reliability, transparency, and integration within the Code_Factory ecosystem. It features dynamic plugin management, distributed messaging, enterprise security, compliance auditing, and automated self-healing.

---

## Key Features

- **Modular Architecture**:  
  - Standardized `Base` class for all components (`initialize`, `shutdown`, `health_check`).
  - Integrates custom plugins and modules, including those generated by Generator or self_fixing_engineer.

- **Asynchronous Core**:  
  - Built on `asyncio` for non-blocking I/O and scalable concurrency.

- **Resilience & Self-Healing**:  
  - Circuit breakers, retry policies, and a dead letter queue (DLQ) for fault tolerance.
  - MetaSupervisor monitors health and triggers auto-recovery.

- **Audit & Explainability**:  
  - Merkle-chain audit records for traceability and compliance.
  - Integration with ExplainableAI for interpretable results.

- **Flexible Backend Support**:  
  - Unified `ArrayBackend` with NumPy, PyTorch, Dask, CuPy, Qiskit, and NengoLoihi support.

- **Plugin Ecosystem**:  
  - Hot-reloading, dependency management, and process isolation for plugins.
  - Marketplace for plugin installation and rating.

- **Distributed Messaging**:  
  - Sharded async message bus with backpressure, encryption, deduplication, and Kafka/Redis bridges.

- **Enterprise Security**:  
  - Encryption at rest/in transit, RBAC/ABAC, audit trails, compliance with SOC2, HIPAA, PCI-DSS, GDPR.

## Core Concepts

### 1. Engine Coordination

- `OmniCoreEngine` and `OmniCoreOmega` manage lifecycle of components (database, bus, plugins).
- Handles init, shutdown, health checks, and system events.

### 2. Messaging

- Sharded message bus
    - Consistent hashing for sharding
    - Retry, circuit breaker, backpressure, encryption, deduplication

### 3. Plugin Management

- Dynamic loading, hot-reload, and rollback with `PluginEventHandler` & `PluginRegistry`
- Use the `@plugin` decorator for registration.
- PluginMarketplace supports install/rating.

### 4. Database Layer

- Async SQLAlchemy ORM for SQLite (dev) / PostgreSQL or Citus (prod)
- FernetEncryption for selected fields
- Secure storage of agent state, audit records

### 5. Auditing & Explainability

- Critical events logged with Merkle chaining for tamper-evidence
- FastAPI endpoints for audit proofs
- ExplainableAI for context and traceability

### 6. Self-Healing

- MetaSupervisor monitors system health and triggers recovery
- Plugin hot-swapping, test synthesis, and integration with self_fixing_engineer

### 7. API & CLI

- FastAPI for plugin management, code fixing, auditing
- CLI for server control, plugin listing, code fixing, metrics, debugging

### 8. Code_Factory Integration

- Generator produces plugins, tests, configurations auto-registered
- self_fixing_engineer provides code refactoring & test generation tools

## Getting Started

### Prerequisites

- Python 3.8+
- Install dependencies:
  ```sh
  pip install -r requirements.txt
  ```
- (Optional) Install additional packages for extended features

### Configuration

Create a `.env` file in your root directory:

```env
DATABASE_URL=sqlite+aiosqlite:///omnicore.db
PLUGIN_DIR=./plugins
LOG_LEVEL=INFO
MESSAGE_BUS_SHARD_COUNT=4
DLQ_MAX_RETRIES=3
```
See [CONFIGURATION.md](CONFIGURATION.md) for details.

### Database Initialization

Tables are auto-created. For migrations:

```sh
alembic init migrations
alembic revision --autogenerate -m "Initial migration"
alembic upgrade head
```

---

### Running the Engine

**Start the server:**
```sh
python -m omnicore_engine.cli serve
```
API: [http://localhost:8000](http://localhost:8000)  
Docs: [http://localhost:8000/docs](http://localhost:8000/docs)

---

### CLI Usage

- Interactive shell:
  ```sh
  python -m omnicore_engine.cli repl
  ```
- List plugins:
  ```sh
  python -m omnicore_engine.cli list-plugins
  ```
- Fix imports in a file:
  ```sh
  python -m omnicore_engine.cli fix-imports path/to/file.py
  ```
- View system metrics:
  ```sh
  python -m omnicore_engine.cli metrics-status
  ```
- Debug system info:
  ```sh
  python -m omnicore_engine.cli debug-info
  ```

---

### Adding a New Plugin

Create a Python file in `PLUGIN_DIR` (e.g., `plugins/my_plugin.py`):

```python
from omnicore_engine.plugin_registry import plugin, PlugInKind

@plugin(kind=PlugInKind.FIX, name="my_plugin", version="1.0.0")
def my_plugin(code: str) -> str:
    return f"Processed {code}"
```
Plugin will be auto-loaded by the PluginEventHandler.  
See [PLUGINS.md](PLUGINS.md) for details.

- **Formatting:** Use `black` for formatting, `flake8` for linting.
- **Type Hints:** Use and validate with `mypy`.
- **Docstrings:** Provide clear docstrings for public APIs.
- **Logging:** Use `structlog` with module binding: `logger.bind(module="MyModule")`.
- **Metrics:** Instrument new components using `metrics.py`.
- **Testing:** Add/maintain unit and integration tests in `tests/` with `pytest`, `pytest-asyncio`.
